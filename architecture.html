NGINX Architecture

NGINX is built around one master process and many other worker processes. Worker processes are event-driven and can handle many connections at once through the use of non-blocking I/O (event loops).

    Master vs. Worker Role

  • Master: reads config, binds ports, manages worker lifecycle.
  • Worker: performs the actual request processing

Why Event Driven?

Compared to a “one thread per connection” model, event-driven workers avoid massive memory and context-switch overhead when handling thousands of clients.

Tip: By default, Nginx spawns about one worker per CPU core — a sensible starting point.
